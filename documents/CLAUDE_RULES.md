# CLAUDE_RULES

Read before implementing anything.

1. Follow Constitution.
2. Respect module boundaries.
3. Never duplicate components.
4. Use shared packages.
5. Build reusable abstractions.
6. Never hardcode colors, roles, permissions.
7. Prefer composition.
8. Add tests.
9. Update documentation.
10. Ask for clarification if architecture is ambiguous.

Definition of Done:
- Builds successfully
- Typed
- Tested
- Documented
- Accessible
- Responsive
- Secure
